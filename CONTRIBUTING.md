# Contributing

Create a focused branch, describe the scientific or software rationale, and include the exact validation command. Changes to preprocessing, splits, labels, or metrics must explain how information leakage is prevented. Do not commit unpublished spectra, manuscripts, credentials, personal paths, or large generated artifacts.

Pull requests should report MATLAB/toolbox versions, tests performed, affected outputs, and whether numerical results changed. Maintainers should require review before merging changes that alter scientific conclusions.
